1 · From a document to sentences

The problem

Your data arrives as paragraphs. But the unit we work with is the sentence. So we need to segment a paragraph into its sentences.

The obvious idea: cut on every full stop. To do that we call a method on the string — some_text.split(".") — using a dot (.) to run a built-in action on that value.

Why this matters

The naive rule looks fine — until it isn’t. Abbreviations and decimals contain dots that aren’t sentence ends.

Bad boundaries corrupt everything downstream. So it’s worth seeing the naive rule fail before we reach for a better tool.

🧪 Let’s try — naive split

paragraph = ("Dr. Smith reviewed the data. The results were clear, "
             "e.g. accuracy rose. Scores went from 3.14 to 9.")

naive = paragraph.split(".")     # cut the string at every "."
for piece in naive:
    print(repr(piece))

Look at the mess: "Dr" (from Dr.), "e" and "g" (from e.g.), "3" and "14" (from 3.14) all split in the wrong places. Sentence boundaries are not just full stops.

2 · Segmentation with a model

Borrow a real tool

Instead of our own rule, we import a purpose-built library, spaCy:

import spacy
nlp = spacy.blank("en")          # a minimal English pipeline
nlp.add_pipe("sentencizer")      # a rule-based sentence splitter — no download
  • import spacy loads the toolbox.
  • The sentencizer knows abbreviations and decimals aren’t sentence ends.

Why this matters

Important

On Day 2, the sentence is the unit you annotate by hand and the unit you feed the LLM. Bad boundaries → bad gold data → misleading scores.

Getting segmentation right is not a side quest — it’s the foundation the whole gold standard sits on.

🧪 Let’s try — the sentencizer

doc = nlp(paragraph)
sentences = list(doc.sents)      # spaCy's sentences, as a list
for s in sentences:
    print(repr(s.text))

Compare to the naive split: Dr. Smith, e.g., and 3.14 stay intact. You now have a real list of sentences — exactly what step 3 loops over.

3 · Run the model over every sentence

Three small pieces of Python

  • for — repeat the same steps for every item in a list.
  • if — react to what comes back (e.g. flag an empty reply).
  • def — package the steps into a reusable function you can call by name.

Together: for each sentence → build a prompt → call the model → tidy the reply.

Why this matters

This little loop is the seed of the pipeline you’ll assemble later in the week.

Annotation, classification, evaluation — all of it is “run this call over a list and collect the results.” Write it once here; reuse the shape all week.

🧪 Let’s try — the loop

for sentence in examples:
    prompt = f"What CEFR level (A1-C2) is this sentence? Answer with just the level. {sentence}"
    reply = generate_text(prompt).strip()
    if reply == "":
        reply = "(no answer)"
    print(sentence, "→", reply)

🧪 Let’s try — wrap it in a function

Same three steps, packaged so you can reuse them:

def ask(sentence):
    """Ask the model for the CEFR level of one sentence; return its reply."""
    prompt = f"What CEFR level (A1-C2) is this sentence? Answer with just the level. {sentence}"
    return generate_text(prompt).strip()

print(ask("Despite the rain, the game went ahead."))

Now labelling a sentence is just ask(...).
